System architecture diagram

Here is the diagram of the Sapient platform hosted in Azure:


  • Azure VM:

    It is the virtual machine from Azure Cloud Platform which gathers APIs, video and webapplication all together in order to process them in one box.


  • Video:

    It is a directory containing video as an input to the APIs.


  • APIs:

    It is an image recognition processing package divided into object, image and face recognition. The output will be the video after being processed by the package.


  • Webapp:

    It is the web application which displays the video which has been processed.




Implementation of finished functionalities

Here is the procedure of how we managed to combine the APIs, Webapp and Azure VM together in the Azure cloud platform:



IMAGE RECOGNITION


For this particular API developed from last year, we used a desktop with gtx1050Ti graphic card to install the API for testing purpose. The reason why we are using such powerful GPU is because of the CUDA required from the tensorflow package and the API requires GPU optimized machine in order to be installed.

Here is the depencies required from installing the Image Recognition API from last year:

  • Tensorflow
  • Openpose
After the API is installed, we will have to configure the API to make sure that the API will capture the video placed in the certain directory as an input and output the video which contains the processed data in another directory. After that, Andreas who was in charge of this part has to pass the information about the API installation to Shiko who is in charge of the system architecture and push the configured API to github.

Here is the information list required for passing the API installation to Shiko:
  • Name and the version of the Softwares have to be installed
  • Order and procedure of installing the Softwares


FACE & OBJECT RECOGNITION

The same procedure of installing and configuring the API will be performed by Shun Shao who is in charge of face and object recognition.


USER INTERFACE

At the prototype stage, user can use web application to view the videos which have been processed by the APIs.
