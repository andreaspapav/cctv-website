Design Principles

When developing the User Interface(UI) of our software we needed to follow the proper design principles that we were taught. We started by researching the UI of the best CCTV monitoring software that exists nowadays. After listing their strengths and weaknesses we knew what we had to include in our own software and what to avoid. We prepared the initial sketches shown further down in the report and handed it out to our classmates. We got the feedback from them in order to produce the second iteration of drawings and finally end up to our prototype version. This last one was produced on a pc using Adobeâ€™s Photoshop in order to achieve the highest quality graphics.

Paper Sketches

Before starting to design the UI of our software on the computer we had to make some initial drawings using pencil and paper to sketch it. These drawings are draft and just give the main idea of how we are planning to make our UI look like.

Initial Sketches

After we processed the input from our classmates, we attempted to draw the initial sketch. The drawing was done by hand quickly just to illustrate the general concept of the UI(User Interface). After spending some time looking on the work we found some changes that needed to be made.


HOMEPAGE

The first drawing for the home page was expected to have all the alerts pop out in a grid pattern as shown on the drawing on the right. The location of the camera, the name and time of the incident are written on the pop-up alert. A blinking alert sign with the level of the emergency is written in the middle to inform the user how important the alert is. The user has the choice to directly call the police by clicking on the call button. Full screen button is also available as well as the "monitored" logo which adds the camera to the monitored page. The menu bar is at the bottom with all the possible pages available.


MAP PAGE

The map page shows the whole area of Camden Borough with the location of each camera. This can be helpful in following a crime when it gets out of the video frame of the camera we are monitoring.


CAMERAS PAGE

Cameras page just shows all the cameras in an alphabetical order, allows the user to search for a specific one or just monitor multiple cameras at the same time. It's the same concept with all the CCTV monitoring UI that exist out there.


EMERGENCY CALLS PAGE

This page is used to list all the emergency calls that the users might need in order to call the police and inform them about the incident. Fire stations numbers, anti-bombing squads and etc.
